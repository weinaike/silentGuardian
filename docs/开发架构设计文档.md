## **1\. 整体架构与阶段性规划总览**

本项目采用轻量级的原生 Android 架构，数据全本地存储。为了保证开发效率与项目的平稳落地，整个系统架构划分为两个主要阶段：

* **Phase 1 (MVP 最小可行性版本)：** 聚焦核心功能的跑通，解决最致命的耗电、误伤真实通话、进程保活以及异常崩溃兜底问题。  
* **Phase 2 (进阶强化版本)：** 聚焦防熊孩子破解、多应用精细化管理以及应用自身的安全防护。

## **2\. Phase 1：MVP 核心架构与功能 (第一阶段)**

### **2.1 进程保活与自启模块 (Battery & Service Manager)**

**解决痛点：** 纯前台服务易被杀；服务被杀后无法自启。

* **\[To AI\] Manifest 声明要求：**  
  * \<uses-permission android:name="android.permission.FOREGROUND\_SERVICE" /\>  
  * \<uses-permission android:name="android.permission.FOREGROUND\_SERVICE\_SPECIAL\_USE" /\> (Target 34 必须)  
  * \<uses-permission android:name="android.permission.RECEIVE\_BOOT\_COMPLETED" /\> (开机自启必须)  
  * Service 注册时必须添加：android:foregroundServiceType="specialUse"  
* **代码强制要求：**  
  * 服务启动时调用 startForeground() 绑定常驻通知。  
  * onStartCommand() 必须返回 **START\_STICKY**，以便系统在资源空闲时自动重启该服务。  
  * 注册静态广播接收器监听 ACTION\_BOOT\_COMPLETED，实现开机自启。

### **2.2 核心监控与防误伤模块 (Core Monitor & Safeguard)**

* **轮询机制优化 (省电策略)：**  
  * 在 Service 中注册 BroadcastReceiver 监听 ACTION\_SCREEN\_ON 和 ACTION\_SCREEN\_OFF。  
  * **息屏时，暂停 2 秒一次的轮询**；亮屏时恢复。  
* **\[To AI\] AppUsageDetector 实现规范 (防止幻觉)：**  
  * 必须使用 UsageStatsManager.queryEvents()，倒序遍历寻找最近的一个 Event.MOVE\_TO\_FOREGROUND 来确定前台应用。绝不能用废弃或模糊的时间段统计 API。  
* **电话防误伤机制 (极其重要)：**  
  * 引入 TelephonyManager。若 callState 处于来电或通话中，关闭 VPN 黑洞恢复网络，同时跳过当前轮询。

### **2.3 UI 实时刷新机制 (Reactive UI)**

* 构建单例对象 FlowManager 包含 val usedTimeFlow \= MutableStateFlow(0)。  
* MonitorService 负责更新 value，MainActivity 在 lifecycleScope 中 collect 此 Flow 并更新 UI 进度条。

### **2.4 MVP 阶段交互与“链式权限申请”**

**解决痛点：** 并发请求权限会导致系统弹窗重叠或回调混乱。

* **\[To AI\] 权限申请时序（必须按此顺序链式调用）：**  
  1. **第一步：** 申请 POST\_NOTIFICATIONS (Android 13+)。  
  2. **第二步：** 申请 PACKAGE\_USAGE\_STATS (使用情况访问权限)。  
  3. **第三步：** 一切就绪后，才显示主开关。当家长打开总控开关时，引导申请 REQUEST\_IGNORE\_BATTERY\_OPTIMIZATIONS (无限制电池优化)。

### **2.5 异常兜底与状态自愈 (Failsafe & Edge Cases \- 极度重要)**

**解决痛点：** 跨日数据不重置导致一直被锁；服务崩溃导致手机麦克风“永久静音”。

* **跨日数据重置闭环：**  
  * 轮询每次累加时间前，先获取当前日期 (如 "20260607")，与 MMKV 中的 LAST\_RECORD\_DATE 对比。  
  * 若日期不一致：强制将 TODAY\_USED\_TIME 归零，并更新 LAST\_RECORD\_DATE。  
* **网络状态泄漏防护（崩溃兜底）：**  
  * 在 MonitorService 的 onDestroy() 生命周期中，**必须**显式关闭 VpnService 并释放连接，恢复正常网络。  
  * 在 Application 初始化时，配置 Thread.setDefaultUncaughtExceptionHandler 全局异常捕获，若发生不可抗力崩溃，强制终止 VPN 进程恢复网络状态。

## **3\. Phase 2：进阶强化架构与功能 (第二阶段)**

### **3.1 防卸载与防篡改模块 (Anti-Tamper Manager)**

* 引入 DeviceAdminReceiver 激活设备管理器，防止直接卸载。  
* 在轮询中检查 XXPermissions.isGranted(PACKAGE\_USAGE\_STATS)，若被关，弹出 SYSTEM\_ALERT\_WINDOW 全屏警告阻断，强制要求重开。

### **3.2 独立应用精细化配额 (Granular Quota System)**

* 将 MMKV 存储升级为 Map\<String, Int\> 格式，针对不同的 App 包名分别进行时长计时与阈值对比。

## **4\. 核心功能逻辑流程图 (供 AI Agent 理解的伪代码)**

// MonitorService.kt 核心轮询 (置于 CoroutineScope.launch(Dispatchers.IO) 中)

override fun onDestroy() {  
    super.onDestroy()  
    // \[To AI\] 绝对生命周期兜底：服务销毁时必须关闭 VPN 恢复网络！  
    VpnManager.stopBlackholeVpn()   
}

// \--- 轮询逻辑 \---  
while (isActive) {  
    delay(2000L)   
      
    if (\!isScreenOn) continue // 挂起，不耗电

    // 1\. 防误伤绝对高优拦截  
    if (telephonyManager.callState \!= TelephonyManager.CALL\_STATE\_IDLE) {  
        VpnManager.stopBlackholeVpn()   
        continue   
    }

    val currentApp \= AppUsageDetector.getForegroundPackageName(context)  
    val managedApps \= MMKV.defaultMMKV().getStringSet("MANAGED\_APPS", emptySet())  
      
    if (managedApps?.contains(currentApp) \== true) {  
        // 2\. 跨日重置校验  
        val todayStr \= getTodayDateString() // 如 "20260607"  
        val lastDate \= MMKV.defaultMMKV().decodeString("LAST\_RECORD\_DATE", "")  
        if (todayStr \!= lastDate) {  
            FlowManager.usedTimeFlow.value \= 0  
            MMKV.defaultMMKV().encode("LAST\_RECORD\_DATE", todayStr)  
        }

        // 3\. 累加与断网干预  
        FlowManager.usedTimeFlow.value \+= 2   
        MMKV.defaultMMKV().encode("TODAY\_USED\_TIME", FlowManager.usedTimeFlow.value)  
          
        val dailyLimit \= MMKV.defaultMMKV().decodeInt("DAILY\_LIMIT", 1800\) // 默认 30 分钟  
        if (FlowManager.usedTimeFlow.value \>= dailyLimit) {  
            if (\!VpnManager.isVpnRunning()) {  
                VpnManager.startBlackholeVpn(currentApp) // 执行局部断网干预，把流量导入黑洞  
            }  
        }  
    } else {  
        // 4\. 切出管控应用  
        if (VpnManager.isVpnRunning()) {  
            VpnManager.stopBlackholeVpn() // 恢复正常网络能力  
        }  
    }  
}
