# **AI语音陪伴防沉迷系统 (代号: SilentGuardian) 项目需求文档**

## **1\. 项目背景与目标**

**背景：** 随着“豆包”、“元宝”等具有极强语音交互能力的 AI 软件普及，儿童极易陷入长时间的连续语音通话中，影响视力与作息。传统强制关闭应用的防沉迷手段过于生硬，容易引发亲子冲突。

**目标：** 开发一款 Android 后台管控应用。通过“定向建立 VpnService 网络黑洞”这一降维打击手段，当儿童使用特定 AI 应用超时后，将其伪装成“局部网络故障/无法连接服务器”的自然现象，引导儿童主动结束使用，实现无痛防沉迷。

## **2\. 系统运转核心要素（技术可行性前提）**

本系统要稳定运转，必须依赖 Android 系统的以下四个核心要素：

1. **应用使用情况访问（UsageStatsManager）：** 用于实时获知当前手机屏幕上正在运行的是哪一款 App（获知前台包名）。  
2. **虚拟网络服务（VpnService）：** 用于执行核心的“建立本地黑洞并重定向特定 App 流量”导致断网的动作。  
3. **前台服务（Foreground Service）：** 确保我们的管控程序能在后台持久运行，不被系统为了省电而“杀掉”。  
4. **开机自启广播（Boot Receiver）：** 确保手机重启后，管控逻辑依然生效。

## **3\. 功能需求说明**

### **3.1 用户端（家长配置界面）**

这是家长进行设置的界面，需要简洁明了，并具备一定的防破解机制。

* **管控开关：** 一键开启/关闭整个后台监控服务。  
* **目标应用选择器：** 读取手机内已安装的应用列表，允许家长勾选需要管控的 App（如“豆包”、“元宝”）。  
* **时长限额设置：** 为勾选的应用设置“每日总可用语音时长”（例如 30 分钟）。  
* **家长锁（可选/进阶）：** 进入该 App 修改配置或关闭服务时，需要输入 PIN 码，防止儿童自行解除。

### **3.2 后台运行端（核心管控服务）**

对用户完全透明，在后台默默执行监控任务。

* **实时状态轮询：** 开启一个定时任务（Timer），每隔 2\~3 秒检查一次当前屏幕前台的应用。  
* **计时器累加：** 如果检测到前台应用是“目标应用”，则将今日已使用时间累加。  
* **动态干预执行：** 当累计时间超过设定的限额时，触发 VpnService 将目标 App 的流量全部导入黑洞导致断网；当离开目标应用时，无论是否超时，立刻关闭 VPN 恢复网络。

初始化：获取今日已使用时长 (Today\_Usage\_Time)，获取每日限额 (Daily\_Limit)

启动定时器 (每 2 秒执行一次):  
    1\. 当前前台应用包名 \= GetForegroundAppPackageName()  
      
    2\. 判断前台应用是否在【管控名单】中：  
       【是管控应用】:  
           Today\_Usage\_Time \+= 2秒 // 累加使用时间  
             
           判断 Today\_Usage\_Time 是否 大于或等于 Daily\_Limit：  
               【已超时】:  
                   if (未处于 VPN 断网状态) {  
                       VpnManager.startBlackhole(目标包名)  // 开启核心干预：建立本地 VPN 黑洞拦截流量

                   }  
               【未超时】:  
                   if (处于 VPN 断网状态) {  
                       VpnManager.stopBlackhole() // 恢复正常网络

## **5\. 关键边界条件与异常处理 (极其重要)**

为了确保系统的安全性和隐蔽性，必须处理好以下边界问题：

1. **“误伤”正常网络/通话：** \* **情景：** 孩子在豆包里超时被断网了，此时刚好奶奶打来微信视频，或者退出豆包刷短视频。  
   * **处理逻辑：** VpnService 使用了 addAllowedApplication 仅定向拦截目标 App，微信等其他 App 流量根本不会进入 VPN；且系统会在 2 秒内立刻关闭 VPN，双保险保证正常网络通讯不受任何影响。  
2. **跨日时间重置：**  
   * **情景：** 到了第二天，限额应该重新计算。  
   * **处理逻辑：** 每次累加时间前，校验当前系统日期。若日期发生更替，自动将 Today\_Usage\_Time 清零。  
3. **防止被系统管家杀后台：**  
   * **处理逻辑：** 必须使用带有持久通知栏的 Foreground Service，并在各大手机（华为、小米、OV）的电池管理中，引导家长手动将本应用设置为“无限制/允许后台高耗电”。  
4. **重启逃逸：**  
   * **处理逻辑：** 注册 android.intent.action.BOOT\_COMPLETED 广播，手机一开机，后台服务立马拉起。并且已使用时间必须持久化存储（如使用 SharedPreferences），不能存放在内存中，防止重启后时间清零。

## **6\. 开发技术栈与所需权限清单**

* **开发语言：** Kotlin (推荐) 或 Java  
* **最小支持版本：** Android 8.0 (API 26\) 以上  
* **Android Manifest 必须声明的权限：**  
  * \<uses-permission android:name="android.permission.PACKAGE\_USAGE\_STATS" /\> (必须引导用户在系统设置的"使用情况访问"中授权)  
  * \<uses-permission android:name="android.permission.BIND\_VPN\_SERVICE" /\> (注册 VPN 服务必须权限)  
  * \<uses-permission android:name="android.permission.FOREGROUND\_SERVICE" /\> (前台服务保活)  
  * \<uses-permission android:name="android.permission.RECEIVE\_BOOT\_COMPLETED" /\> (接收开机广播)
